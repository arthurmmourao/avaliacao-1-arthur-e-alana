# **Avaliação 1 - Paradigmas da Programação**

**1) Quais as diferenças percebidas entre os dois paradigmas?**

As diferenças são que no paradigma funcional foca na transformação dos dados, recebendo um dado e entrega uma saída tranformada. Já no paradigma imperativo é feito um passo-a-passo, interagindo com o usuário.

**2) Quais as vantagens e desvantagens de cada abordagem?**

Paradigma imperativo
* Vantagens: é bem intuitivo, você manda e computador obedece.
* Desvantagens: é mais fácil de se perder quando utilizamos muitas variáveis.

Paradigma funcional
* Vantagens: é seguro, fácil de testar e não altera nada externo.
* Desvantagens: A sua lógica é mais difícil de ser pensada.

**3) Qual abordagem foi mais fácil/dificil e por quê?**

Funcional é mais fácil pois sua lógica é direta, no início parece mais difícil por envolver lógica matemática.

Imperativo é mais difícil, porque apesar de ser mais fácil de ser escrito ele é mais difícil de se manter quando o código cresce pois os dados mudam.

**4) Qual o impacto na legitibilidade e manutenção de código?**

Na legitibilidade, o código, está limpo e organizado, sem muita poluição visual. contudo, algumas ferramentas podem causar dúvidas em quem não tem conhecimento sobre.

Na manutenção, a lógica está boa, a separação das tarefas(responsabilidades) está boa. Se ocorrer algum problema não será difícil de encontrá-lo.
